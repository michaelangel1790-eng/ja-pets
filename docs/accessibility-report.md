# דוח נגישות - JACUZZI (עדכון)

**תאריך:** 01/05/2026  
**מטרה:** שיפור נגישות בהתאם להנחיות **WCAG 2.1 רמה AA** ולתקן הישראלי **ת״י 5568** (כהשראה; אין כאן אישור משפטי מלא).

---

## מה בוצע (סיכום טכני)

### ליבת נגישות ומקלדת

- קישור **«דלג לתוכן הראשי»** נשמר ב־`layout.tsx`.
- **מסגרת פוקוס ברורה** ל־`:focus-visible` על קישורים, כפתורים, שדות, `[tabindex]`, ותוויות לשוניות (`role="tab"`).
- **לשוניות המידע** (`info-tabs`): ניווט מקלדת ב־**חיצים ימין/שמאלה**, **Home** / **End** ברשימת הטאבים, עם עדכון כתובת ופוקוס לטאב הנבחר.
- הוסרו **`outline: none`** מהפס התחתון הקבוע כדי לאפשר פוקוס נראה.

### מבנה סמנטי וכותרות

- **דף הבית:** `h1` מוסתר ויזואלית (`.visually-hidden`) + `footer` לקישורים המשפטיים.
- **עמודי מדיניות / תנאים / ביטולים / הצהרה / 404:** עטיפה ב־`PageWithChrome` - ניווט עליון, כפתור נגישות צף ופס יצירת קשר כמו בדף הבית.
- מקטעים בעמודי מדיניות: כותרות משנה מעוצבות כ־**`h2`** במקום פסקאות מודגשות בלבד.

### טפסים

- **השארת פרטים:** לכל שדה יש `label` + `id`, קבוצת אישורים ב־`fieldset`/`legend`, הודעות שגיאה עם **`role="alert"`**, **`aria-live="assertive"`**, גבול צבעוני (לא רק אדום טקסט), ו־`aria-invalid` מותנה.
- **טופס ביקורת (בטאב המלצות):** שדות עם תוויות מפורשות, אימות לפני שליחה, והודעות משוב עם `role="alert"` או `role="status"` לפי סוג ההודעה.

### גלריה ולייטבוקס

- ESC סוגר, **מלכודת פוקוס**, החזרת פוקוס לכפתור שנפתח ממנו, חיצים עם `aria-label`, תיאורי `alt` לתמונות, כפתורי זום עם `aria-label`.

### כפתור נגישות (תוספת)

- תפריט: הגדלה/הקטנה, ניגודיות, הדגשת קישורים, הפחתת תנועה, איפוס - כפי שביקשת; **אינו מחליף** התאמות נגישות אמיתיות באתר.

### ניגודיות

- הוחמר גוון **כותרות משנה** (`.section-subtitle`) וטקסטים משניים במספר נקודות (`neutral-300` → `neutral-200` היכן שסומן).

### הצהרת נגישות

- נוסף הבהרה שלא ניתן להצהיר על **עמידה משפטית מלאה** ללא ביקורת מקצועית חיצונית ובדיקות ידניות.
- תאריך עדכון ההצהרה עודכן.

### בדיקות אוטומטיות

- ריצת **`npm run test:a11y`**: סריקת **axe-core** עם תגיות **WCAG 2.1 AA** (דרך Playwright) לנתיבים: `/`, `/lead-details`, `/accessibility-statement`, `/privacy-policy`, `/terms-of-use`, `/cancellation-policy`.  
- **תוצאה אחרונה:** 6 בדיקות עברו ללא הפרות שזוהו על ידי axe בצירוף התגיות הנ״ל.

## מה דורש בדיקה ידנית / מקצועית (מומלץ)

הכלים האוטומטיים **אינם מכסים** את כל דרישות ת״י 5568 / WCAG. מומלץ בפרט:

1. **קורא מסך** - NVDA / JAWS / VoiceOver: זרימת טאבים, טפסים, לשוניות, לייטבוקס, פס קבוע תחתון.
2. **תוכן דינמי** - הודעות מנהל, העלאות גלריה, מחיקות (אם בשימוש).
3. **שפות וכיווניות** - עברית RTL, מספרים וטלפונים.
4. **בדיקת ניגודיות ידנית** על רקעים מדורגים / שקיפויות / תמונות.
5. **ביקורת משפטית** חיצונית אם נדרש אישור רשמי לעמידה בתקן.
